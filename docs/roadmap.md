# BarPOS — Roadmap & следующие шаги

> Дата: апрель 2026. Анализ сделан на основе текущего состояния кода, CLAUDE.md и docs/.

---

## Что уже сделано (краткий итог)

Проект в хорошей форме: полноценный JWT + RBAC, весь базовый CRUD (столы, заказы, меню, склад, пользователи, роли), PDF-чек, статистика за день, аудит-лог (частичный), i18n (EN/RU/KA), настраиваемая валюта, тесты на SQLite. По факту — работающий минимальный POS.

---

## Что из планов можно убрать / понизить приоритет

| Пункт | Решение | Причина |
|---|---|---|
| **Celery + Redis** | Убрать из планов | Нет фоновых задач которые их требуют; добавит операционную сложность без реальной пользы на масштабе одного бара |
| **WebSocket / real-time** | Понизить приоритет | Текущий polling 30 сек покрывает 95% реальных кейсов; WS — значительная инфраструктура ради небольшого UX-выигрыша |
| **Рецепты / Recipe Components** | Понизить приоритет | Сложная фича (новые таблицы, логика списания), нужна только если бар сам делает коктейли из ингредиентов; большинство баров работают с позицией как единицей |
| **Кухонный экран (cook view)** | Убрать из scope | Выходит за рамки бара; для ресторана — отдельный продукт |
| **ESC/POS (print to printer)** | Оставить в backlog | Зависит от конкретного железа; PDF-скачивание решает 80% кейсов |

## Баги и долг которые надо закрыть прямо сейчас

Это не фичи, а broken things. Нужно сделать в первую очередь.

### 1. CORS настроен на Streamlit-порт (8501), а не на React-клиент
**Где:** `app/main.py` — `allow_origins`  
**Что:** в браузере React-клиент получает CORS-ошибку на API-запросы в dev-режиме  
**Объём:** 1 строка в `main.py` + ENV-переменная `CORS_ORIGINS` в `.env.example`  
**Сложность:** тривиально

### 2. Аудит-лог не пишет события по столам, заказам, позициям и складу
**Где:** `app/api/routes_v1.py` — вызовы `_audit()` есть только для auth + user/role событий  
**Что:** если нужен полный журнал (кто создал стол, кто добавил заказ, кто скорректировал склад) — его нет  
**Объём:** добавить `_audit()` в ~8 маршрутов; 15–20 строк  
**Сложность:** низкая

### 3. Переименование стола — есть API, нет UI
**Где:** `client/src/pages/TableDetail.jsx` — статичный `<h1>` с именем стола  
**Что:** бэкенд `PATCH /tables/{id}` поддерживает изменение имени; фронт — нет  
**Объём:** добавить inline-редактирование (иконка карандаша + input + save) в один компонент  
**Сложность:** низкая

### 4. `docs/architecture.md` устарел
**Где:** файл  
**Что:** React SPA помечен как "Planned", хотя давно реализован; схема API не актуальна  
**Объём:** обновить таблицы статусов и добавить реальные эндпоинты  
**Сложность:** нет кода, просто документация

### 5. `client_legacy/` (Streamlit) путает и ломает CORS
**Где:** `client_legacy/`, `docker-compose.yml`  
**Что:** старый Streamlit-клиент больше не нужен; он является причиной CORS-настройки на порт 8501  
**Объём:** удалить директорию и сервис из docker-compose  
**Сложность:** тривиально; нужно убедиться что ничего больше не зависит

### 6. Лицензия не выбрана
**Где:** `README.md` — `(**licensename_TODO**)`  
**Что:** проект open-source, но без лицензии — юридически "все права защищены"  
**Объём:** выбрать (MIT / Apache 2.0) и добавить `LICENSE` + исправить README  
**Сложность:** нет кода, организационное решение

## Следующие фичи — план по приоритетам

### Приоритет 1 — Делаем сразу (малый объём, высокая ценность)

#### A. Пагинация на list-эндпоинтах
**Зачем:** без неё `GET /tables/?status=Closed` за несколько месяцев работы будет возвращать тысячи записей  
**Где меняется:**
- `app/schemas/` — добавить `PaginatedResponse[T]` обёртку
- `app/api/routes_v1.py` — `skip` + `limit` query params на `/tables/`, `/items/`, `/audit/events` (на `/orders/` уже не нужно — это sub-ресурс)
- `client/src/pages/` — Audit.jsx уже имеет `limit` selector; Tables.jsx, Menu.jsx, Stock.jsx добавить кнопку «загрузить ещё» или простую нумерацию  

**Объём:** средний (бэк — ~30 строк, фронт — ~50 строк на страницу)  
**Сложность:** низкая

#### B. Профиль пользователя / смена пароля
**Зачем:** сейчас смена пароля — только через admin. Базовый UX для любого staffа  
**Где меняется:**
- `app/api/routes_v1.py` — `PUT /users/me` (смена пароля с проверкой старого)
- `client/src/pages/Profile.jsx` — новая страница, форма смены пароля
- `client/src/components/Layout.jsx` — ссылка на профиль в sidebar (имя пользователя → /profile)

**Объём:** малый (~40 строк бэк, ~80 строк фронт)  
**Сложность:** низкая

#### C. Расширенный диапазон дат в статистике
**Зачем:** `GET /stats/daily` возвращает только один день; нельзя посмотреть неделю или месяц  
**Где меняется:**
- `app/services/table_service.py` — `get_daily_stats()` уже принимает `date`; добавить `date_from / date_to`
- `app/api/routes_v1.py` — обновить query params
- `client/src/pages/Stats.jsx` — заменить одиночный date picker на range picker (start/end date)

**Объём:** малый (бэк ~20 строк, фронт ~30 строк)  
**Сложность:** низкая

### Приоритет 2 — Следующий спринт (средний объём, бизнес-ценность)

#### D. Управление сменами (Shifts)
**Зачем:** без смен нельзя понять кто из персонала работал, считать выручку по кассиру, контролировать кассу  
**Где меняется:**
- `app/models/models.py` — новая таблица `Shift` (`id, user_id, opened_at, closed_at, opening_cash, closing_cash, notes`)
- `app/migrations/` — новая Alembic-миграция
- `app/api/routes_v1.py` — `POST /shifts/` (открыть), `POST /shifts/{id}/close`, `GET /shifts/`
- `app/services/` — `shift_service.py`
- `client/src/pages/Shifts.jsx` — новая страница: текущая смена, кнопка открыть/закрыть, история
- `client/src/i18n/locales/` — новые ключи

**Объём:** средний (~150 строк бэк, ~200 строк фронт)  
**Сложность:** средняя

#### E. Оплата и способ оплаты
**Зачем:** сейчас нет записи о том как платил гость (наличные/карта/вкладка). Без этого нельзя делать Z-отчёт  
**Где меняется:**
- `app/models/models.py` — добавить `payment_method` enum (`cash/card/tab`) и `Payment` запись при закрытии стола (или поле в `Table`)
- `app/api/routes_v1.py` — `POST /tables/{id}/close` принимает `payment_method`
- `app/schemas/` — обновить `TableCloseRequest`
- `client/src/pages/TableDetail.jsx` — при закрытии стола модальное окно с выбором способа оплаты
- `client/src/pages/Stats.jsx` — разбивка выручки по способу оплаты

**Объём:** средний (~80 строк бэк, ~100 строк фронт)  
**Сложность:** средняя

#### F. Уведомления о низком остатке склада
**Зачем:** сейчас Stock.jsx красит строку при ≤3 ед., но это видно только если открыть страницу склада  
**Где меняется:**
- `client/src/components/Layout.jsx` — badge с количеством low-stock позиций на иконке Склада в сайдбаре (данные берутся из уже существующего `/items/` с фильтром)
- `client/src/pages/Stock.jsx` — уже есть счётчик; просто сделать его заметнее  
- Опционально: `app/api/routes_v1.py` добавить `GET /items/low-stock` для быстрого запроса

**Объём:** малый-средний (~20 строк бэк, ~40 строк фронт)  
**Сложность:** низкая

### Приоритет 3 — Backlog (нужно, но можно подождать)

#### G. HTTPS в production-деплое
**Зачем:** сейчас всё HTTP; JWT и пароли летят в открытом виде на продакшне  
**Где меняется:**
- `docker-compose.yml` — добавить nginx-сервис с SSL-терминацией
- `nginx.conf` — новый файл конфигурации (proxy_pass на :8000 и :8080)
- `docs/` — инструкция по Let's Encrypt / самоподписанному сертификату  

**Объём:** средний (конфигурация, не код)  
**Сложность:** средняя (зависит от деплой-окружения)

#### H. Скидки и комп (Discounts)
**Зачем:** бармен должен иметь возможность убрать позицию с чека или дать скидку  
**Где меняется:**
- `app/models/models.py` — `Order.discount` (%) или `Table.discount_total`
- `app/api/routes_v1.py` — `PATCH /tables/{id}/orders/{oid}` поддерживает `discount`
- `client/src/pages/TableDetail.jsx` — поле скидки на строке заказа

**Объём:** средний  
**Сложность:** средняя (нужно пересчитывать total при закрытии)

#### I. Исторические отчёты
**Зачем:** владелец хочет видеть тренды — топ позиций, выручку по месяцам  
**Где меняется:**
- `app/api/routes_v1.py` — новые эндпоинты `/stats/top-items`, `/stats/revenue-by-period`
- `client/src/pages/Stats.jsx` — дополнительные графики (нужна библиотека; Recharts уже используется)

**Объём:** средний  
**Сложность:** средняя
